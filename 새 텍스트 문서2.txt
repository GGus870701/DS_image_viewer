# 윈도우용 이미지 & CAD 뷰어 구현 플랜

무료 라이브러리만을 활용하여 BMP, JPG, TIF, DXF, DWG 파일을 지원하는 윈도우 뷰어를 개발하기 위한 종합적인 구현 계획입니다.

## **1. 프로젝트 개요**

### **목표**
- 다양한 이미지 포맷(BMP, JPG, TIF) 및 CAD 포맷(DXF, DWG) 통합 뷰어 개발
- CAD 파일 전용 기능: 레이어 ON/OFF, 배경색 흰색/검은색 변환
- 완전 무료 라이브러리만 사용하여 라이선스 비용 제로화
- 윈도우 네이티브 성능 최적화

### **핵심 요구사항**
- **파일 지원**: BMP, JPG, TIF, DXF, DWG
- **CAD 전용 기능**: 레이어 제어, 배경색 토글
- **성능**: 대용량 파일 부드러운 렌더링
- **비용**: 무료 라이브러리만 사용

## **2. 기술 스택 선정**

### **개발 언어 및 프레임워크 비교**

**옵션 A: C# + WPF (강력 권장)**
- **장점**
  - 윈도우 네이티브 성능 및 하드웨어 가속 지원
  - 복잡한 CAD 렌더링에 최적화된 DrawingVisual/StreamGeometry
  - 풍부한 .NET 생태계 및 안정적인 메모리 관리
  - Visual Studio 통합 개발 환경
- **단점**
  - 윈도우 전용 플랫폼
  - WPF 학습 곡선 존재
- **라이선스**: .NET 6+ MIT 라이선스 (상업적 사용 완전 자유)

**옵션 B: Python + PySide6**
- **장점**
  - 빠른 프로토타이핑 및 풍부한 라이브러리
  - 크로스 플랫폼 지원 가능성
- **단점**
  - 실행 속도 및 메모리 사용량 이슈
  - 복잡한 패키징 및 배포 과정
- **라이선스**: PySide6 LGPL (상업적 사용 가능, 동적 링크 조건)

**최종 권장**: **C# + WPF** - 성능, 안정성, 라이선스 측면에서 최적
