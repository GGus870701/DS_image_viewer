# DS Image Viewer - 다중 이미지 일괄 변환 기능 구현 계획

본 계획서는 사용자 피드백을 반영하여, 탐색기 우클릭 및 메인 뷰어에서 호출 가능한 **고급 이미지 일괄 변환(리사이즈, 회전, 포맷 변경 등)** 기능에 대한 구체적인 아키텍처와 UI 요구사항을 정리한 문서입니다.

## 1. 아키텍처 및 진입점 설계

### 1.1 CLI 인자 및 다중 인스턴스 제어 (`main.py`)
- **다중 파일 선택 대응**: Windows 탐색기에서 여러 파일을 선택하고 우클릭 메뉴를 실행하면, 각 파일마다 프로세스가 실행되는 문제가 발생할 수 있습니다. 이를 방지하기 위해 `QLocalServer` 또는 `QSharedMemory`를 활용한 **Single Instance 패턴**을 적용합니다. 
- 첫 번째 프로세스가 메인 윈도우(ConvertWindow)를 띄우고 서버가 되며, 이후 동시에 실행된 프로세스들은 서버로 파일 경로만 전달하고 종료되도록 하여, 하나의 UI 목록에 모든 파일이 담기도록 구현합니다.
- **실행 인자**: `main.py --convert <file1> <file2> ...`

### 1.2 탐색기 우클릭 메뉴 등록 (`core/registry_mgr.py`)
- **메뉴명**: `"DS뷰어로 변환"`
- **등록 키**: `HKCU\Software\Classes\SystemFileAssociations\image\shell\ds_convert`
- 배포 시 폴더(onedir) 방식으로 빌드되는 것을 감안하여, 현재 실행 중인 `.exe` 경로를 올바르게 레지스트리에 등록/해제하는 기능을 제공합니다.

### 1.3 메인 뷰어 연동 (`ui/main_window.py`)
- 툴바 및 메뉴바에 **[DS뷰어로 변환]** 버튼을 추가하여, 뷰어 사용 중에도 현재 보고 있는 이미지나 선택한 폴더의 이미지들을 변환 창으로 넘길 수 있도록 연결합니다.

## 2. 변환 윈도우 UI/UX 설계 (`ui/convert_window.py`)

변환 윈도우는 복잡한 설정을 직관적으로 제어할 수 있도록 좌측(파일 목록)과 우측(설정 패널)으로 나뉘어 구성됩니다.

### 2.1 좌측 패널: 변환 대상 파일 목록
- **QListWidget / QTableWidget** 사용
- 탐색기에서 넘어온 파일 목록을 표시.
- 드래그 앤 드롭을 지원하여 추가 파일이나 폴더를 쉽게 목록에 넣을 수 있도록 지원.
- 파일 삭제(Delete 키) 및 초기화 버튼 포함.

### 2.2 우측 패널: 변환 설정 (5대 카테고리)
설정 옵션들은 `QGroupBox` 또는 `QToolBox`를 활용하여 5가지 카테고리로 묶어서 제공합니다.

#### [1] 회전 (Rotation)
- 라디오 버튼 또는 콤보박스:
  - 회전 안함
  - 좌로 90도
  - 우로 90도
  - 180도
  - **EXIF 정보를 사용해서 자동으로 (디폴트)**

#### [2] 크기 조절 (Resize)
- **변환 모드 (드롭다운)**: 크기 변경 안함, **비율 유지 (디폴트)**, 폭 맞춤, 높이 맞춤, 여백 붙이기, 여백 자르기, 꽉차게 늘리기.
- **크기 입력 (인풋 박스)**: 
  - 폭(Width): 기본값 `640` px
  - 높이(Height): 기본값 `480` px
  - *단위 선택기(px / %) 추가하여 퍼센트 비율 조정도 가능하게 구현.*

#### [3] 저장 이미지 포맷 (Format & Quality)
- **포맷 (콤보박스)**: **기존 형식 유지 (디폴트)**, JPG, PNG
- **화질 (인풋 박스/슬라이더)**: 기본값 `80` (JPG 등 손실 압축 포맷에 적용)
- **EXIF 보존 (체크박스)**: EXIF 정보 보존 여부 선택 (기본: 선택)
- **멀티코어 처리 (스핀박스)**: `자동 (디폴트)` ~ 시스템 최대 코어 수까지 선택 가능하여 일괄 변환 시 퍼포먼스 극대화. (파이썬 `multiprocessing` 또는 `ProcessPoolExecutor` 활용)

#### [4] 파일 저장 경로 (Save Location)
- **저장 위치 (라디오 버튼)**:
  - 같은 위치에 저장
  - 특정 폴더에 저장 (우측에 경로 인풋 박스 및 '찾아보기' 버튼 활성화)
  - **다음 이름의 하위 폴더에 저장 (디폴트)**: 기본값 `output`. 원본 사진이 있는 위치 하부에 해당 이름의 폴더를 생성하여 저장.

#### [5] 파일 이름 설정 (Naming Strategy)
- **중복 처리 (콤보박스)**: **이름 바꾸기 (예: 파일명(2)) (디폴트)**, 덮어쓰기.
- **접두어 (Prefix)**: 체크박스 및 붙일 이름 인풋 박스 (저장될 파일명 앞에 텍스트 추가).

### 2.3 하단 액션 버튼
- **설정 초기화**: 모든 옵션을 위 디폴트 상태로 되돌리는 버튼.
- **변환 시작 (Start)**: 설정된 옵션을 바탕으로 백그라운드 워커를 통해 변환 작업 시작.
- **닫기 (Close)**

## 3. 핵심 변환 로직 (`core/image_converter.py`)

- UI 스레드가 멈추지 않도록 **멀티프로세싱 워커(Worker)** 분리.
- 이미지 리사이징 시 고품질 필터(Bicubic/Lanczos) 사용.
- 사용자가 선택한 모드(여백 붙이기, 여백 자르기 등)에 따라 QImage를 생성/조작하는 정밀 수학 계산 로직 구현.
- EXIF 데이터 추출 후, 변환된 결과물에 다시 삽입(piexif 또는 pillow 라이브러리 연계 고려).

---

## 4. 검증 계획 (Verification Plan)
1. **탐색기 다중 선택 테스트**: 탐색기에서 사진 10장을 선택하고 "DS뷰어로 변환"을 눌렀을 때, 10개의 창이 뜨지 않고 **단 1개의 창**에 10개의 파일이 정상 로드되는지 검증 (IPC 서버 동작 확인).
2. **리사이즈 모드 테스트**: 각 크기 조절 옵션(여백 자르기, 폭 맞춤 등)이 의도한 수학적 비율대로 정확히 잘리고 크기가 변하는지 결과물 확인.
3. **EXIF 및 회전 연계**: 스마트폰으로 세로로 찍은 사진(EXIF 회전 태그 존재)이 "EXIF 정보 사용(자동)" 옵션 적용 시 똑바로 서 있는 상태로 리사이징되는지 확인.
4. **성능 모니터링**: 코어 개수를 최대로 할당했을 때의 처리 속도 측정 및 UI 응답성(Freeze 방지) 확인.
