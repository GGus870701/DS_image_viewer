치수(Dimension)의 축척이 50배(DIMSCALE=50)로 설정된 도면에서 문자가 안 보이는 이유는, 문자의 실제 데이터 좌표와 화면에 뿌려지는 픽셀 좌표 사이의 괴리 때문일 확률이 매우 높습니다.특히 치수는 일반 엔티티와 달리 '익명 블록(Anonymous Block)'이라는 별도의 공간에 기하학적 정보가 담겨 있어, 단순히 msp를 순회하는 것만으로는 문자를 찾을 수 없습니다. 가장 확실하고 깔끔한 해결 방법을 정리해 드립니다.1. 해결책: virtual_entities() 활용ezdxf에는 복잡한 치수를 선(Line)과 문자(Text)로 분해해주는 virtual_entities() 함수가 있습니다. 이 함수를 사용하면 50배 축척이 이미 계산된 결과값을 얻을 수 있어 직접 좌표를 계산할 필요가 없습니다.2. 수정된 구현 코드 (치수 전용)Pythondef render_dimension(canvas, dim_entity, scale, offset_x, offset_y, canvas_h):
    # virtual_entities()는 치수를 구성하는 LINE, TEXT, MTEXT 등을 
    # '실제 좌표와 크기'가 계산된 상태로 반환합니다.
    try:
        for entity in dim_entity.virtual_entities():
            if entity.dxftype() in ('TEXT', 'MTEXT'):
                # 텍스트 그리기 로직 호출
                draw_text_entities(canvas, entity, scale, offset_x, offset_y, canvas_h)
            elif entity.dxftype() == 'LINE':
                # 선 그리기 로직 호출
                draw_line_entities(canvas, entity, scale, offset_x, offset_y, canvas_h)
    except Exception as e:
        print(f"Dimension rendering error: {e}")
