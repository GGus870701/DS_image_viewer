1. 위치 불일치 원인: 텍스트 기준점 (Alignment)
CAD의 TEXT 엔티티는 작성자가 어떤 정렬(좌측, 중앙, 우측 등)을 썼느냐에 따라 좌표를 읽는 곳이 다릅니다.

TEXT의 기본 삽입점은 dxf.insert (보통 좌측 하단)입니다.

하지만 정렬(Alignment)이 지정된 경우, 실제 기준점은 dxf.align_point에 저장됩니다. (예: 중앙 정렬 시)

해결 코드 (기준점 보정):

Python
def get_text_position(entity):
    if entity.dxftype() == 'TEXT':
        halign = entity.dxf.halign
        valign = entity.dxf.valign
        
        # halign이 0(기본값, 왼쪽)이고 valign이 0(기본값, 기준선)일 때만 insert 사용
        # 그 외의 정렬 값이 있다면 align_point를 사용해야 정확한 위치가 됨
        if halign == 0 and valign == 0:
            return entity.dxf.insert
        else:
            # align_point가 없으면 insert 사용 (방어적 코드)
            return entity.dxf.get('align_point', entity.dxf.insert)
            
    elif entity.dxftype() == 'MTEXT':
        # MTEXT는 기본적으로 insert만 사용 (단, MTEXT의 attachment_point 속성에 따라 다름)
        return entity.dxf.insert
2. 크기 불일치 원인: 도면 단위 vs 화면 픽셀 (Scale)
CAD의 글자 높이(dxf.height 또는 dxf.char_height)는 도면의 실제 단위(mm, inch 등)입니다. 도면 전체를 화면에 맞추기 위해 곱한 배율(scale)을 글자 높이에도 정확히 곱해야 합니다.

주의점: Tkinter의 폰트 크기는 '포인트(pt)' 단위이거나 시스템 설정에 종속되므로, CAD의 높이에 scale을 곱한 값과 시각적으로 차이가 날 수 있습니다.

해결 코드 (크기 보정):

Python
# 1. 폰트 높이 계산 (픽셀 기준)
# MTEXT는 char_height, TEXT는 height 속성을 가짐
cad_height = entity.dxf.char_height if entity.dxftype() == 'MTEXT' else entity.dxf.height
pixel_height = cad_height * scale 

# 2. Tkinter 폰트 크기로 변환 (적절한 보정 계수 적용)
# 시스템에 따라 다르지만 보통 pixel 높이를 음수로 넣으면 픽셀 단위 크기로 지정됨 (Tkinter 특징)
tk_font_size = int(pixel_height) * -1 

# 너무 작아 안 보이는 경우 방지
if abs(tk_font_size) < 5: 
    tk_font_size = -5 

tk_font = ("Arial", tk_font_size)
3. Tkinter 캔버스에 그리기 (Anchor 설정)
CAD의 기본 정렬(좌측 하단)을 Tkinter의 기준점인 anchor와 맞춰야 합니다.
Tkinter는 기본적으로 중앙(anchor="center")에 글씨를 찍으려고 합니다.

Python
# 위치 좌표 가져오기
pos = get_text_position(entity)

# 화면 좌표로 변환 (Y축 반전)
screen_x = (pos.x * scale) + offset_x
screen_y = canvas_h - ((pos.y * scale) + offset_y)

# 텍스트 회전 (CAD 각도는 반시계방향, Tkinter도 angle 파라미터로 지원 가능)
angle = entity.dxf.get('rotation', 0) if entity.dxftype() == 'TEXT' else entity.dxf.get('rotation', 0)

# MTEXT의 경우 attachment_point에 따라 anchor가 복잡하지만, 우선은 sw(좌하단)로 테스트
anchor_type = "sw" 

canvas.create_text(
    screen_x, screen_y,
    text=content,
    fill="white",
    font=tk_font,
    anchor=anchor_type,
    angle=angle  # Tkinter 8.6 이상부터 angle 지원
)
MTEXT (여러 줄 텍스트)의 특이점
MTEXT는 단순히 한 점에 텍스트가 있는 게 아니라, 너비 박스가 지정된 단락 형태입니다.
ezdxf의 .plain_text()로 글자를 뽑아도, 엔티티의 attachment_point (1~9 사이의 숫자, 1=TopLeft, 5=Middle, 9=BottomRight)에 따라 화면에 찍히는 위치가 다릅니다.

완벽한 일치를 원하신다면 아래 매핑 테이블을 참고해 Tkinter의 anchor에 연결해야 합니다.

attachment_point 1 (좌상단) -> anchor="nw"

attachment_point 2 (상단중앙) -> anchor="n"

attachment_point 5 (정중앙) -> anchor="center"

attachment_point 7 (좌하단) -> anchor="sw"

요약 및 적용 순서
위치(Position): TEXT는 align_point를 우선 확인.

크기(Size): cad_height * scale 값에 -1을 곱해 Tkinter 폰트 사이즈(픽셀 지정 방식)로 적용.

기준점(Anchor): TEXT는 보통 "sw", MTEXT는 attachment_point에 따라 조정.
