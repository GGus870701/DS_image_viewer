### B. 메모리 사용량 최적화 (Memory Management)
*   **Problem**: DXF 도면 열람 시 RAM 사용량이 400MB 이상으로 치솟아 시스템 리소스를 과도하게 점유함.
*   **Cause**: 
    *   초기 썸네일 해상도가 4000px로 설정되어 한 장당 약 64MB의 비압축 비트맵 메모리 소모.
    *   내비게이터 캐시가 무제한으로 쌓여 여러 파일을 열수록 메모리 점유율이 선형적으로 증가.
    *   PIL 이미지 객체 및 회전 처리된 비트맵들이 명시적으로 해제되지 않고 메모리에 잔류.
*   **Solution**: 
    1.  **해상도 하향**: 초기 DXF 렌더링 해상도를 4000px에서 2048px로 조정 (픽셀 수 기준 4배 절감, 메모리 약 75% 감소).
    2.  **캐시 제한 (LRU)**: 내비게이터 썸네일 캐시를 최대 5개로 제한하여 오래된 데이터는 자동 폐기.
    3.  **명시적 리소스 해제**: 이미지 로드 시 기존 이미지의 `close()`를 호출하고 `gc.collect()`를 강제 수행하여 즉각적인 메모리 회수 유도.
    4.  **회전 캐시 초기화**: 새 이미지를 불러올 때 이전 이미지의 회전/반전 처리 비트맵 캐시를 명시적으로 `None` 처리.
