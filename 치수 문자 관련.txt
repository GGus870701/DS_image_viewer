CAD의 Dimension(치수) 엔티티는 단순히 선과 숫자로 이루어진 게 아니라, 내부적으로 매우 복잡한 구조를 가지고 있어서 좌표나 축척이 틀어지기 쉽습니다. 특히 ezdxf를 사용할 때 치수가 작게 나오거나 엉뚱한 위치에 있다면 다음 3가지 핵심 요소를 점검하고 수정해야 합니다.

1. 치수의 실체: 익명 블록(Anonymous Block) 활용
CAD에서 치수는 사실 하나의 '블록'입니다. DIMENSION 엔티티 자체의 데이터를 계산하려고 하면 화살표 크기, 문자 오프셋, 축척 등을 일일이 계산해야 하므로 거의 불가능에 가깝습니다.

해결책: 치수 엔티티가 가리키는 geometry 블록을 찾아 그 내부의 선(Line)과 문자(Text)를 그려야 합니다.

치수 엔티티에는 *D1, *D2와 같은 이름의 블록이 연결되어 있습니다. 이 블록 안에는 이미 축척이 적용된 완성된 형태의 기하학적 정보가 들어 있습니다.

Python
# ezdxf에서 치수 블록의 데이터를 가져오는 법
dim = msp.query('DIMENSION').first
# 치수를 구성하는 실제 도형들이 담긴 블록 이름을 가져옴
block_name = dim.dxf.geometry 
dim_block = doc.blocks.get(block_name)

for entity in dim_block:
    # 이 엔티티들을 일반 LINE, TEXT처럼 처리해서 그리면 축척이 맞음
    if entity.dxftype() == 'LINE':
        draw_line(entity)
2. 전역 축척(DIMSCALE)과 선형 축척(DIMLFAC)
치수 데이터에는 두 가지 주요 축척 변수가 있습니다. 블록을 직접 사용하지 않고 수동으로 그리신다면 이 값을 반드시 곱해줘야 합니다.

DIMSCALE (전역 축척): 화살표 크기, 문자 높이 등 '표시용' 요소들의 크기를 결정합니다. (예: 1:100 도면이면 보통 100)

DIMLFAC (길이 축척): 실제 측정된 길이에 곱해지는 값입니다. 모델 공간과 종이 공간의 단위가 다를 때 사용합니다.

Python
# ezdxf에서 스타일별 축척 가져오기
dim_style = doc.dimstyles.get(dim.dxf.dimstyle)
scale = dim_style.dxf.dimscale # 전역 축척
linear_factor = dim_style.dxf.dimlfac # 선형 축척
3. Tkinter 캔버스의 좌표 변환(Viewport) 문제
CAD 파일의 단위(mm, m)와 캔버스의 픽셀(Pixel) 단위를 매칭시키는 전역 변환 함수가 없으면 치수가 눈에 보이지 않을 만큼 작거나 화면 밖으로 나갑니다.

구현 팁:
전체 도면의 경계(Bounding Box)를 먼저 구한 뒤, 화면 크기에 맞춰 Scale 값을 계산하는 클래스를 하나 만드시는 게 좋습니다.

Python
class Viewport:
    def __init__(self, canvas_width, canvas_height, dxf_extents):
        # 도면의 실제 크기와 캔버스 크기를 비교해 배율 계산
        dxf_w = dxf_extents.width
        dxf_h = dxf_extents.height
        self.scale = min(canvas_width / dxf_w, canvas_height / dxf_h) * 0.9
        self.offset_x = -dxf_extents.min_x * self.scale
        self.offset_y = -dxf_extents.min_y * self.scale

    def transform(self, x, y):
        # CAD 좌표 -> Tkinter 화면 좌표 (Y축 반전 포함)
        screen_x = (x * self.scale) + self.offset_x
        screen_y = canvas_height - ((y * self.scale) + self.offset_y)
        return screen_x, screen_y
요약: 치수가 안 맞을 때의 체크리스트
치수가 너무 작나요? -> DIMSCALE이 적용 안 된 것입니다. 치수 블록(*D...)을 루프 돌려 그리세요.

치수 위치가 엉뚱한가요? -> DIMENSION 엔티티의 defpoint가 아닌 블록 내부 엔티티의 절대 좌표를 사용했는지 확인하세요.

문자가 깨지거나 없나요? -> 치수 블록 안의 MTEXT나 TEXT 엔티티를 파싱하고 있는지 확인하세요.

가장 추천하는 방법은 ezdxf에서 제공하는 ezdxf.addons.drawing 모듈을 참고하는 것입니다. 비록 Tkinter 전용은 아니지만, 내부적으로 엔티티를 어떻게 분해(Flatten)해서 그리는지 로직이 잘 구현되어 있습니다.
